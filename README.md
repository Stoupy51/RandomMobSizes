This datapack provides a funny game experience by randomizing every mob sizes in your world! üé≤ üéÆ<br>
The default scale is 0.75-1.25 but can be configured as you want with the command `/function random_mob_sizes:config` ‚öôÔ∏è

The datapack is optimized, meaning it do not add any lag to your world/server! ‚ö°Ô∏è

You can also configure per-mob behavior to fine‚Äëtune size ranges and health scaling for specific entities. üéØ

## Configuration ‚öôÔ∏è

Run the helper command to configure the pack:

- ‚ñ∂Ô∏è Execute `/function random_mob_sizes:config`
- üí¨ This prints clickable messages in chat that prefill commands you can edit and send.

What you can configure (from the in-game helper):

- üìè Value range for sizes: 0.0625 ‚Äì 16.0
- üåê Default global range: min_size = 0.75, max_size = 1.25
- ‚ù§Ô∏è Health scaling toggle: whether mob health scales with their size (enabled by default)
- üèÉ Speed scaling toggle: whether mob movement speed scales with their size (disabled by default)
- ‚öîÔ∏è Damage scaling toggle: whether mob attack damage scales with their size (disabled by default)

Global settings examples üõ†Ô∏è:

- ‚ûñüìè Set minimum size
  - `/data modify storage random_mob_sizes:config min_size set value 0.75`
- ‚ûïüìè Set maximum size
  - `/data modify storage random_mob_sizes:config max_size set value 1.25`
- üîÅ‚ù§Ô∏è Enable/disable health scaling
  - `/data modify storage random_mob_sizes:config health_scale set value false`
  - Use `true` to enable, `false` to disable.
- üîÅüèÉ Enable/disable speed scaling
  - `/data modify storage random_mob_sizes:config speed_scale set value true`
  - Use `true` to enable, `false` to disable.
- üîÅ‚öîÔ∏è Enable/disable damage scaling
  - `/data modify storage random_mob_sizes:config damage_scale set value true`
  - Use `true` to enable, `false` to disable.

Per-mob overrides (optional) üêæ:

- üß¨ You can override the global settings for a specific mob type. Provide any subset of keys; omitted keys fall back to the global values.
- üêî Example for chickens:
  - `/data modify storage random_mob_sizes:config mobs."minecraft:chicken" set value {min_size:0.75,max_size:1.25,health_scale:true,speed_scale:false,damage_scale:false}`
- üîÅ Replace `minecraft:chicken` with the desired entity ID (e.g., `minecraft:zombie`).

üí° Tip: You can re-run `/function random_mob_sizes:config` at any time to show the clickable helpers again.

## Technical / Compatibility üîß
Mobs concerned by the datapack are:
- mobs with a "Brain" NBT üß†
- not players üë§
- not armor stands üõ°Ô∏è
- not entities having the following Tags: `random_mob_sizes.checked`, `smithed.entity`, `smithed.strict`, `global.ignore` üè∑Ô∏è

Meaning modded mobs can be concerned! üéØ

If you want to revert all loaded entities to their original size, you can type `/execute as @e run attribute @s scale base reset`.
